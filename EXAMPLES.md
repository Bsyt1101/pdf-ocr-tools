# 使用示例

## 示例1：基本使用

### 场景
处理xxx单位的等保测评文档。

### 操作步骤

1. **准备PDF文件**
   ```
   xxx单位.pdf（18页）
   ```

2. **运行程序**
   ```bash
   python main.py
   ```

3. **输入信息**
   ```
   请输入 PDF 文件路径: xxx单位.pdf
   请输入项目名称（可选）: JDB项目编号-xxx单位
   请输入系统名称（可选）: 教务系统
   请输入输出目录（默认为当前目录）: ./output
   ```

4. **处理过程**
   ```
   正在拆分 PDF: xxx单位.pdf
   总页数: 18
     拆分第 1 页 → page_001.pdf
     拆分第 2 页 → page_002.pdf
     ...

   开始识别文档类型...

   处理第 1 页:
       检测到扫描件（文本<50字符），使用阿里云 OCR...
     提取文本: 中科实数 文档编号：JDB项目编号-FXGZS-01 风险告知书...
     识别类型: 风险告知书 (项目级)

   处理第 2 页:
       检测到扫描件（文本<50字符），使用阿里云 OCR...
     提取文本: 中科实数 文件编号：JDB项目编号-CPSQS-01 现场测评授权书...
     识别类型: 测评授权书 (项目级)

   ...
   ```

5. **查看结果摘要**
   ```
   ============================================================
   处理结果摘要
   ============================================================
   项目名称: JDB项目编号-xxx单位
   系统名称: 教务系统
   总页数: 18

   文档类型统计:
     测评授权书: 1 页
     测评调研表: 1 页
     风险告知书: 1 页
     入离场确认书: 3 页
     ...

   详细列表:
     第  1 页 → JDB项目编号-xxx单位-风险告知书.pdf
              类型: 风险告知书
              文件夹: 风险告知书

     第  2 页 → JDB项目编号-xxx单位-测评授权书.pdf
              类型: 测评授权书
              文件夹: 测评授权书
   ...
   ```

6. **确认归档**
   ```
   是否执行文件归档？(y/n): y

   开始归档文件...
     page_001.pdf → 风险告知书/JDB项目编号-xxx单位-风险告知书.pdf
     page_002.pdf → 测评授权书/JDB项目编号-xxx单位-测评授权书.pdf
     ...

   归档完成！
   ```

7. **查看结果**
   ```bash
   ls -R output/
   ```

   输出：
   ```
   output/:
   测评授权书  测评调研表  风险告知书  入离场确认书  ...

   output/测评授权书:
   JDB项目编号-xxx单位-测评授权书.pdf

   output/测评调研表:
   教务系统-测评调研表.pdf

   output/风险告知书:
   JDB项目编号-xxx单位-风险告知书.pdf
   ...
   ```

## 示例2：处理多个系统

### 场景
一个项目包含3个系统的测评文档。

### 操作步骤

**第一次运行 - 系统1**
```
请输入 PDF 文件路径: 系统1文档.pdf
请输入项目名称（可选）: JDB项目编号-xxx单位
请输入系统名称（可选）: 门户网站系统
请输入输出目录（默认为当前目录）: ./output
```

**第二次运行 - 系统2**
```
请输入 PDF 文件路径: 系统2文档.pdf
请输入项目名称（可选）: JDB项目编号-xxx单位
请输入系统名称（可选）: 内部办公系统
请输入输出目录（默认为当前目录）: ./output
```

**第三次运行 - 系统3**
```
请输入 PDF 文件路径: 系统3文档.pdf
请输入项目名称（可选）: JDB项目编号-xxx单位
请输入系统名称（可选）: 数据库系统
请输入输出目录（默认为当前目录）: ./output

### 最终结果

```

├── 测评调研表/

│   ├── 内部办公系统-测评调研表.pdf
│   └── 数据库系统-测评调研表.pdf
├── 测评方案确认书/
│   ├── 门户网站系统-测评方案确认书.pdf
│   ├── 内部办公系统-测评方案确认书.pdf
│   └── 数据库系统-测评方案确认书.pdf
└── ...

## 示例3：处理扫描件和电子文档混合PDF

### 场景
PDF中既有扫描件，也有带文本层的电子文档。

### 处理过程

```
处理第 1 页:
  提取文本: 中科实数 文档编号：JDB项目编号-FXGZS-01...  # 有文本层，直接提取
  识别类型: 风险告知书 (项目级)

处理第 2 页:
    检测到扫描件（文本<50字符），使用阿里云 OCR...  # 扫描件，使用OCR
  提取文本: 中科实数 文件编号：JDB项目编号-CPSQS-01...
  识别类型: 测评授权书 (项目级)

处理第 3 页:
  提取文本: 系统情况调查表确认签字...  # 有文本层，直接提取
  识别类型: 测评调研表 (系统级)
```

**说明**：程序会自动判断每一页是否需要OCR，优化处理速度。

## 示例5：自定义分类规则

### 场景
需要识别新的文档类型"安全责任书"。

### 操作步骤

1. **编辑 doc_classifier.py**

   在第29行附近添加新规则：
   ```python
   CLASSIFICATION_RULES = [
       # 项目级文档
       (["授权书", "测评"], "测评授权书", False),
       (["风险告知"], "风险告知书", False),
       (["安全责任"], "安全责任书", False),  # 新增规则
       # ... 其他规则
   ]
   ```

2. **添加文件夹映射**

   在第14行附近添加：
   ```python
   DOC_TYPE_TO_FOLDER = {
       "测评授权书": "测评授权书",
       "风险告知书": "风险告知书",
       "安全责任书": "安全责任书",  # 新增映射
       # ... 其他映射
   }
   ```

3. **测试分类器**
   ```bash
   python doc_classifier.py
   ```

4. **重新处理PDF**
   ```bash
   python main.py
   ```

### 结果

```
处理第 X 页:
  提取文本: ...安全责任书...
  识别类型: 安全责任书 (项目级)

output/
├── 安全责任书/
│   └── JDB项目编号-xxx单位-安全责任书.pdf
└── ...
```

## 示例6：环境变量配置

### Linux/Mac

```bash
# 临时设置（当前终端有效）
export ALIYUN_OCR_APPCODE="your_aliyun_appcode_here"
python main.py

# 永久设置（添加到 ~/.bashrc 或 ~/.zshrc）
echo 'export ALIYUN_OCR_APPCODE="your_aliyun_appcode_here"' >> ~/.zshrc
source ~/.zshrc
```

### Windows

```cmd
# CMD（临时）
set ALIYUN_OCR_APPCODE=your_aliyun_appcode_here
python main.py

# PowerShell（临时）
$env:ALIYUN_OCR_APPCODE="your_aliyun_appcode_here"
python main.py

# 永久设置（系统环境变量）
# 1. 右键"此电脑" → 属性 → 高级系统设置
# 2. 环境变量 → 新建
# 3. 变量名：ALIYUN_OCR_APPCODE
# 4. 变量值：your_aliyun_appcode_here
```

## 常见使用技巧

### 技巧1：快速测试

只处理前几页，快速验证配置：

修改 `pdf_processor.py` 第151行：
```python
for page_num in range(min(3, len(doc))):  # 只处理前3页
```

### 技巧2：调整OCR质量

提高DPI以获得更好的识别效果：
```python
processor = PDFProcessor(dpi=300)  # 默认200
```

### 技巧3：批量处理

创建简单的批处理脚本：
```bash
#!/bin/bash
for pdf in *.pdf; do
    echo "处理: $pdf"
    python main.py <<EOF
$pdf
JDB项目编号-xxx单位
教务系统
./output
y
EOF
done
```

### 技巧4：查看详细日志

在代码中添加更多调试信息，或将输出重定向到文件：
```bash
python main.py 2>&1 | tee process.log
```

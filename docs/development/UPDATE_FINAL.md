# 命名规则最终确认

## ✅ 当前命名规则

**项目级文档**: `项目名-文档类型.pdf`
**系统级文档**: `系统名-文档类型.pdf`

## 📝 命名示例

### 项目级文档
```
JDB25300-广东时代传媒-现场测评授权书.pdf
JDB25300-广东时代传媒-风险告知书.pdf
JDB25300-广东时代传媒-启动会议记录.pdf
JDB25300-广东时代传媒-保密承诺书.pdf
```

### 系统级文档
```
门户网站系统-情况调查表.pdf
内部办公系统-情况调查表.pdf
数据库系统-测评方案评审记录.pdf
```

## 🔍 文件编号的作用

文件编号（如 `JDB25300-CPSQS-01`）**仅用于文档类型识别**，不用于文件命名。

### 识别流程

```
PDF内容: 文档编号：JDB25300-CPSQS-01
         现场测评授权书

识别步骤:
1. 提取文件编号: JDB25300-CPSQS-01
2. 解析类型码: CPSQS → 现场测评授权书
3. 判断级别: 项目级
4. 生成文件名: JDB25300-广东时代传媒-现场测评授权书.pdf
   (使用项目名，不使用文件编号)
```

## ✨ 优势

1. **可读性强** - 文件名包含完整信息
2. **易于管理** - 按项目/系统分类清晰
3. **兼容性好** - 与现有命名习惯一致

## 🎯 使用方法

无需任何特殊配置，按正常流程使用即可：

```bash
python main.py

请输入 PDF 文件路径: 广东时代传媒.pdf
请输入项目名称（可选）: JDB25300-广东时代传媒
请输入系统名称（可选）: 门户网站系统
请输入输出目录（默认为当前目录）: ./output
```

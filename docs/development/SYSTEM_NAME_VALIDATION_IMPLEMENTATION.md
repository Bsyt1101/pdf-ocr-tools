## 验证逻辑

### 验证时机

1. **模糊匹配时**：当通过文本匹配从实施单中获取系统名称时
2. **内容提取时**：当从页面内容中提取系统名称时

### 验证规则

比较以下信息：
- 文件编号中的系统序号（如 `JDB25300-CPSQS-02` 中的 `02`）
- 实施单中的系统名称映射（如 `{1: "门户网站系统", 2: "内部办公系统"}`）
- 页面提取或匹配到的系统名称

### 智能比较算法

```python
# 去除常见后缀进行核心名称比较
extracted_core = current_system_name.replace('系统', '').replace('平台', '').strip()
expected_core = expected_name.replace('系统', '').replace('平台', '').strip()

# 只有核心名称完全不匹配时才报警
if extracted_core not in expected_core and expected_core not in extracted_core:
    print("⚠️  警告：系统名称不一致！")
```

## 使用效果

### 优势
1. **自动检测**：无需人工检查，自动发现不一致
2. **及时提醒**：处理过程中实时显示警告信息
3. **智能比较**：避免因"系统"/"平台"等后缀差异导致的误报
4. **详细提示**：清晰显示不一致的具体信息，便于排查

### 适用场景
1. 文件编号写错（系统序号错误）
2. 实施单中系统名称缺失或错误
3. OCR 识别错误导致系统名称不准确
4. 页面内容与文件编号不匹配

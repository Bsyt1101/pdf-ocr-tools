## ✨ 新功能

### 1. 基于文件编号的智能命名

工具现在可以自动识别PDF文档中的文件编号，并使用文件编号作为文件名。

**文件编号格式**: `JDB项目编号-文档类型码-系统编号`

**示例**:
```
文档内容: 文档编号：JDB25300-CPSQS-01
         现场测评授权书

识别结果: JDB25300-CPSQS-01.pdf
```

### 2. 15种文档类型简拼映射

添加了标准的文档类型简拼对照表：

| 简拼 | 文档类型 |
|------|---------|
| CPSQS | 现场测评授权书 |
| FXGZS | 风险告知书 |
| DCB | 情况调查表 |
| FAFS | 测评方案评审记录 |
| FACR | 测评方案确认书 |
| SCHY | 启动会议记录 |
| MCHY | 末次会议记录 |
| CPJG | 测评结果记录 |
| CPWT | 测评问题列表 |
| SMQR | 漏洞扫描记录 |
| CTCS | 渗透测试记录 |
| WDJJ | 现场接收归还文档清单 |
| RLCQR | 入离场确认单 |
| YSB | 项目验收评估表 |
| BMCNS | 保密承诺书 |

### 3. 智能命名优先级

```
1. 检查文档中是否有文件编号
   ↓
   有 → 使用文件编号命名 ✓ (新功能)
   ↓
   无 → 使用项目名/系统名命名 (原有功能)
```

## 📝 使用方法

### 无需任何修改

如果你的PDF文档中已经包含标准的文件编号，工具会自动识别并使用。

**示例PDF内容**:
```
文档编号：JDB25300-CPSQS-01
现场测评授权书
授权方：广东时代传媒集团有限公司
```

**自动生成文件名**: `JDB25300-CPSQS-01.pdf`

### 兼容旧格式

如果PDF中没有文件编号，工具会自动使用原有的命名方式，完全向后兼容。

## 💡 使用建议

### 推荐做法

1. **标准化文档**: 在文档页眉添加标准文件编号
   ```
   文档编号：JDB项目编号-类型码-系统编号
   ```

2. **统一格式**: 使用统一的文件编号格式
   ```
   JDB25300-CPSQS-01  ✓ 推荐
   JDB25300CPSQS01    ✗ 不支持
   ```

3. **清晰标识**: 使用"文档编号："或"文件编号："前缀
   ```
   文档编号：JDB25300-CPSQS-01  ✓
   编号：JDB25300-CPSQS-01      ✓
   JDB25300-CPSQS-01            ✓ (也支持)
   ```

## ❓ 常见问题

### Q: 旧版本处理的文件需要重新处理吗？

A: 不需要。新旧两种命名方式都是有效的。

### Q: 如果文档中没有文件编号会怎样？

A: 工具会自动使用原有的命名方式（项目名-文档类型），完全兼容。

### Q: 可以自定义文件编号格式吗？

A: 可以。修改 `doc_classifier.py` 中的正则表达式即可。详见 [NAMING_RULES.md](NAMING_RULES.md)。

### Q: 如何添加新的文档类型简拼？

A: 编辑 `doc_classifier.py` 的 `DOC_TYPE_CODE_MAP` 字典。详见 [NAMING_RULES.md](NAMING_RULES.md)。

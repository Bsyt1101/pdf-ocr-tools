# 更新日志

## [1.7.5] - 2026-03-22

### 新增
- ✨ **百度飞桨 API 参数优化** - 优化 PaddleOCR-VL-1.5 请求参数，大幅提升识别质量
  - `useLayoutDetection: False` — 关闭版面区域分割，改用纯 OCR 全图识别，不再遗漏页眉文件编号
  - `promptLabel: 'ocr'` — VL 模型纯文字识别模式，配合关闭版面检测
  - `useDocOrientationClassify: True` — 开启方向矫正（0°/90°/180°/270°）
  - `useDocUnwarping: True` — 开启扭曲矫正（褶皱/倾斜）
  - `visualize: False` — 不返回可视化结果图，减少延迟
- ✨ **Tesseract 页眉文件编号兜底** - OCR 引擎未提取到文件编号时，自动用 PyMuPDF 内置 Tesseract 补充识别
  - 对所有 OCR 引擎生效，仅在主 OCR 未提取到文件编号时触发
  - 确保文件编号不丢失，下游系统名称映射和交叉验证能正常执行

### 修复
- 🐛 **修复关键词分类误匹配** - 收紧宽泛关键词规则，避免非目标文档被错误分类
  - `["验收评估"]` → `["项目验收"]`：匹配"项目验收表"/"项目验收内容"等 OCR 输出
  - `["渗透测试"]` → `[("渗透测试", "确认")]`：排除勾选框 `☐渗透测试` 误命中
  - `["漏洞扫描"]` → `[("漏洞扫描", "确认")]`：同理收紧
  - 解决项目验收评估表被误分类为渗透测试记录签字确认表的问题

## [1.7.4] - 2026-03-22

### 新增
- ✨ **编号冲突自动检测与纠正** - 文件编号指向的系统与页面实际内容不一致时自动纠正
  - 交叉验证：方式1（文件编号映射）获取系统名称后，对比页面内容中实际出现的系统名称
  - 自动纠正：发现不一致时自动使用内容中的系统名称，避免错误合并和命名
  - 冲突告警汇总：处理结束后统一展示所有编号冲突，提示用户核实源文档
  - 适用场景：两个系统共用同一编号（如 BGFS-01 同时用于两个系统的报告复审记录）
- ✨ **OCR 连字符变体统一** - `_normalize_ocr_text()` 增加连字符标准化
  - 将 `—`（em dash）、`–`（en dash）、`−`（数学减号）、`ー`（日文长音）、`一`（中文一）统一为 ASCII `-`
  - 解决云 OCR 把文件编号中 `-` 识别为其他字符导致编号提取失败的问题
- ✨ **测评结果记录签字页关键词兜底** - 新增 `("测评层面", "测评人员")` 关键词规则
  - 当文件编号无法识别时，通过页面表头内容识别测评结果记录签字页

### 修复
- 🐛 **修复系统级文档无系统名称时被错误合并** - 关键词匹配路径中，系统级文档无法确定系统归属时，每页独立不合并
  - 之前：两个不同系统的同类文档（如渗透测试签字确认表）在关键词匹配时合并键相同，被错误合并
  - 修复后：合并键包含页码，确保不同系统的页面不会被合并
- 🐛 **修复同编号不同系统文档被合并** - `_merge_pages` 增加系统级文档冲突检测
  - 合并前检查同一编号组内的系统名称，发现不同系统则自动拆分为独立组
- 🐛 **修复 OCR 预览仍显示 HTML 标签** - 文本标准化移至预览打印之前执行

## [1.7.3] - 2026-03-22

### 新增
- ✨ **统一 OCR 文本标准化架构** - 所有 OCR 引擎输出统一经过 `_normalize_ocr_text()` 处理
  - 去除 HTML 标签（百度飞桨云服务返回的 `<div>`、`<table>` 等）
  - 去除 LaTeX 公式（部分模型产生的无意义公式如 `\( \text{EICAT} \)`）
  - 去除 Markdown 图片语法
  - 合并多余空格和空行
  - 确保所有引擎的分类、编号提取、命名逻辑完全一致
- ✨ **异步 API 结果下载重试** - `_fetch_async_result` 增加 3 次重试，应对临时 SSL/网络错误

### 修复
- 🐛 **修复系统级文档降级导致命名和验证失效** - 关键词匹配时不再降级 doc_type 名称
  - 之前：关键词匹配到系统级文档时，名称降级为项目级（如"渗透测试报告"），且 `is_system_level=False`
  - 导致：系统名称无法替换（"渗透测试报告"中没有"xxx系统"前缀），一致性验证分支被跳过
  - 修复后：保留原始系统级类型名（如"xxx系统-渗透测试记录签字确认表"）和 `is_system_level=True`
- 🐛 **修复未识别页面被错误合并** - 未识别页面的合并键改为包含页码（`unknown-pageN`），每页独立保存
  - 之前：所有未识别页面合并键都是 `unknown`，导致不同序号的未分类文档被合并成一个 PDF
- 🐛 **修复文件夹映射缺失导致崩溃** - 补充"渗透测试报告"和"报告评审记录"的文件夹映射
  - `folder_keyword` 为 None 时归入"未分类"文件夹，不再抛出 TypeError

## [1.5.0] - 2026-03-05

### 新增
- ✨ **多OCR引擎支持** - 支持阿里云OCR、DeepSeek-OCR（PaddleOCR-VL-1.5）和Claude OCR
  - 阿里云OCR：速度快（1-2秒/页），适合标准文档
  - DeepSeek-OCR：基于PaddleOCR-VL-1.5，精度高，适合复杂布局
  - Claude OCR：基于Claude Sonnet 4.6，多模态理解能力强
  - 通过 `ocr_engine` 参数灵活切换
  - 支持环境变量自动检测（ALIYUN_OCR_APPCODE、SILICONFLOW_API_KEY、ANTHROPIC_API_KEY）
- ✨ **OCR引擎优先级** - 自动选择最佳OCR引擎
  - 优先级：阿里云 OCR > PaddleOCR-VL-1.5 > Claude Sonnet 4.6
  - 根据环境变量自动选择
  - 默认使用阿里云 OCR
- ✨ **文件编号OCR错误自动修正** - 修正常见的OCR识别错误
  - JD8/JD9 → JDB（自动去掉第一位数字8或9）
  - 示例：JD925136 → JDB25136
- ✨ **SMOR类型映射支持** - 支持OCR错误修正
  - SMOR → SMQR（OCR将Q误识别为O）
- ✨ **未分类文件夹自动创建** - 自动创建"未分类"文件夹存储未识别文档

### 优化
- 🔧 **提高OCR DPI到300** - 从200提升到300，提高识别质量
- 🔧 **Claude模型升级** - 从claude-3-5-sonnet-20241022升级到claude-sonnet-4-6
- 🔧 **OCR引擎显示名称自动获取** - 添加get_display_name()方法，动态显示引擎名称
- 🔧 **Claude OCR提示词优化** - 减少幻觉问题
  - 添加temperature=0.0确保确定性输出
  - 优化提示词，强调只识别实际可见文字

### 修复
- 🐛 **修复文件编号识别错误** - 修正JD925136被识别为JD925136而非JDB25136的问题
- 🐛 **修复OCR引擎显示名称硬编码** - 改为动态获取，支持所有引擎

- 🔧 **文件编号识别增强** - 改进正则表达式，支持更多格式
  - 允许空格：`JDB25300 - CPJG - 01`
  - 大小写不敏感：`jdb25300-cpjg-01`
  - 允许换行符

### 合并逻辑
```
检测文件编号
  ↓
相同编号？ → 是 → 分组
  ↓
检查页面间隔
  ↓
间隔 > 5？ → 是 → 询问用户
  ↓           ↓ 否
合并页面    自动合并
  ↓
生成单个PDF文件
```

### 示例
```
输入：
  第8页：JDB25300-CPJG-01（测评结果记录）
  第9页：JDB25300-CPJG-01（测评结果记录）

输出：
  门户网站系统-测评结果记录签字页.pdf（包含2页）
```

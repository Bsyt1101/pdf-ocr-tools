# 文件命名规则说明

## 📋 命名规则概述

工具使用**项目名/系统名**作为文件命名前缀。

**项目级文档**: `项目名-文档类型.pdf`
**系统级文档**: `系统名-文档类型.pdf`

**重要**: 文件名不添加页码后缀，即使同类型有多页文档。

---

## 🔍 文件编号识别

### 识别用途

工具会识别PDF中的文件编号（格式：`JDB项目编号-类型码-系统编号`），用于：
- ✅ **文档类型识别** - 通过类型码快速识别文档类型
- ❌ **不用于文件命名** - 文件名使用项目名/系统名

### 识别格式

```
JDB数字-字母-数字
```

**示例**:
- `JDB项目编号-CPSQS-01`
- `文档编号：JDB项目编号-FXGZS-01`
- `文件编号：JDB项目编号-DCB-02`

## 🎯 命名规则

```
1. 识别文档类型
   ├─ 有文件编号 → 通过类型码识别
   └─ 无文件编号 → 通过关键词识别
   ↓
2. 判断文档级别
   ├─ 系统级 → 使用系统名
   └─ 项目级 → 使用项目名
   ↓
3. 生成文件名
   ├─ 系统级: 系统名-文档类型.pdf
   └─ 项目级: 项目名-文档类型.pdf
```

## ⚙️ 配置说明

### 添加新的文档类型简拼

编辑 `doc_classifier.py` 第16-32行：

```python
DOC_TYPE_CODE_MAP = {
    "CPSQS": "现场测评授权书",
    "FXGZS": "风险告知书",
    # 添加新类型
    "NEWCODE": "新文档类型",
}
```

### 修改文件编号格式

如果你的文件编号格式不同，修改 `doc_classifier.py` 第110行的正则表达式：

```python
# 当前格式：JDB数字-字母-数字
pattern = r'(JDB\d+)-([A-Z]+)-(\d+)'

# 示例：修改为其他格式
pattern = r'(项目\d+)-([A-Z]+)-(\d+)'
```

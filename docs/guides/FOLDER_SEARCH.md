## 🔄 重要变更

### 从创建文件夹改为搜索文件夹

**旧逻辑（v1.2.0及之前）**:
- 工具会在输出目录下自动创建文件夹
- 例如：创建 `output/测评授权书/`、`output/风险告知书/` 等

**新逻辑（v1.3.0）**:
- 工具不再创建文件夹
- 在PDF文件所在目录下搜索已存在的文件夹
- 根据文件夹名称包含关键词来匹配

## 💡 使用示例

### 场景1：标准文件夹命名

**PDF文件位置**:
```
/项目目录/
├── 项目文档.pdf
├── 01测评授权书/
├── 02风险告知书/
├── 03测评调研表/
└── 13现场接收归还文档清单（Word+PDF）/
```

**处理结果**:
- 现场测评授权书 → 匹配到 `01测评授权书/`
- 风险告知书 → 匹配到 `02风险告知书/`
- 情况调查表 → 匹配到 `03测评调研表/`
- 现场接收归还文档清单 → 匹配到 `13现场接收归还文档清单（Word+PDF）/`

### 场景2：未找到匹配文件夹

**PDF文件位置**:
```
/项目目录/
├── 项目文档.pdf
├── 01测评授权书/
└── 02风险告知书/
```

**处理结果**:
- 现场测评授权书 → ✓ 匹配到 `01测评授权书/`
- 风险告知书 → ✓ 匹配到 `02风险告知书/`
- 情况调查表 → ✗ 警告：未找到包含 '测评调研表' 的文件夹，跳过

## 🔍 搜索逻辑

### 1. 确定搜索目录
```python
# PDF文件路径
pdf_path = "/项目目录/项目文档.pdf"

# 搜索目录 = PDF文件所在目录
search_dir = "/项目目录/"
```

### 2. 遍历文件夹
```python
for folder in search_dir:
    if folder.is_dir():
        # 检查文件夹名称是否包含关键词
        if "测评授权书" in folder.name:
            # 匹配成功
            return folder
```

### 3. 移动文件
```python
# 如果找到匹配的文件夹
target_folder = "/项目目录/01测评授权书/"
target_file = target_folder / "现场测评授权书.pdf"

# 移动文件
source.rename(target_file)
```

## ⚠️ 注意事项

### 1. 文件夹必须预先存在
- 工具不会创建新文件夹
- 如果找不到匹配的文件夹，文件会被跳过
- 建议在运行工具前先创建好所需的文件夹

### 2. 文件夹命名要求
- 文件夹名称必须**包含**关键词
- 关键词匹配是**子串匹配**，不需要完全相同
- 例如：`13现场接收归还文档清单（Word+PDF）` 包含 `现场接收归还文档清单`，可以匹配

### 3. 多个匹配的处理
- 如果有多个文件夹包含同一个关键词，会使用第一个匹配的文件夹
- 建议每个关键词只对应一个文件夹

### 4. 文件名冲突处理
- 如果目标文件夹中已存在同名文件
- 会自动添加后缀：`文件名_1.pdf`、`文件名_2.pdf` 等

## 🔧 配置文件夹关键词

如果需要修改文件夹关键词，编辑 `doc_classifier.py` 中的 `DOC_TYPE_TO_FOLDER` 映射：

```python
DOC_TYPE_TO_FOLDER = {
    "现场测评授权书": "测评授权书",  # 关键词
    "风险告知书": "风险告知书",      # 关键词
    # ...
}
```

## 📊 处理流程

```
1. 拆分PDF
   ↓
2. 识别文档类型
   ↓
3. 确定文件夹关键词
   ↓
4. 在PDF所在目录搜索文件夹
   ↓
5. 找到匹配 → 移动文件
   未找到 → 跳过并警告
```

## 🎯 优势

### 1. 灵活性
- 支持任意文件夹命名格式
- 只要包含关键词即可匹配

### 2. 兼容性
- 兼容现有的文件夹结构
- 不会破坏已有的文件组织

### 3. 安全性
- 不会创建新文件夹
- 不会覆盖现有文件（自动重命名）

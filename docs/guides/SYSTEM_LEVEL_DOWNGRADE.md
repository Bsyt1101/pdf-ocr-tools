# 系统级文档自动降级

**版本**: v1.6.3
**更新日期**: 2026-03-07

---

## 📖 功能概述

当文件编号中的类型简拼无法识别时，系统会通过文本关键词匹配文档类型。如果匹配到的是系统级文档（如"xxx系统-报告初审记录"），会自动降级为项目级命名（如"报告评审记录"），避免因文件编号错误导致的命名混乱。

---

## 🎯 使用场景

### 场景1：文件编号类型简拼错误
```
文件编号：JDB25300-BGF-01
文本内容：网络安全等级保护测评报告评审记录
```

**问题**：`BGF` 不在类型简拼映射表中，无法识别

**处理**：
1. 文件编号提取失败，返回 `None`
2. 进入关键词匹配逻辑
3. 匹配到"报告"+"评审" → 原本是"xxx系统-报告初审记录"
4. 自动降级为"报告评审记录"（项目级命名）

### 场景2：文件编号缺失
```
文本内容：提供门户网站系统的系统情况调查表
```

**问题**：文档中没有文件编号

**处理**：
1. 文件编号提取失败
2. 关键词匹配到"情况调查表" → 原本是"xxx系统-情况调查表确认签字"
3. 自动降级为"测评调研表"（项目级命名）

## 🔍 工作原理

### 分类逻辑流程

```
1. 尝试从文件编号提取文档类型
   ├─ 成功 → 使用文件编号中的类型（系统级命名）
   └─ 失败 → 进入关键词匹配

2. 关键词匹配文档类型
   ├─ 匹配到项目级文档 → 直接使用项目级命名
   └─ 匹配到系统级文档 → 自动降级为项目级命名

3. 降级处理
   ├─ 查找 SYSTEM_TO_PROJECT_LEVEL 映射表
   ├─ 获取对应的项目级命名（文件夹名称）
   └─ 返回项目级命名，is_system_level=False
```

### 代码实现

**位置**：`src/doc_classifier.py` 第 270-290 行

```python
# 如果没有文件编号，使用关键词匹配
# 注意：此时匹配到系统级文档应降级为项目级命名（因为没有明确的系统序号）
for keywords, doc_type, is_system_level in self.CLASSIFICATION_RULES:
    # 检查关键词匹配
    if match:
        # 如果匹配到系统级文档，降级为项目级命名
        if is_system_level and doc_type in self.SYSTEM_TO_PROJECT_LEVEL:
            project_level_name = self.SYSTEM_TO_PROJECT_LEVEL[doc_type]
            return project_level_name, False  # 返回项目级命名
        return doc_type, is_system_level
```

## 💡 设计理念

### 为什么需要降级？

1. **避免命名混乱**
   - 系统级文档需要系统名称前缀（如"消费者报道官网系统-报告初审记录"）
   - 如果文件编号无法识别，就无法确定系统序号
   - 强行使用"xxx系统-"前缀会导致命名不规范

2. **保持一致性**
   - 降级后的命名与文件夹名称一致
   - 便于用户理解和管理
   - 避免出现"xxx系统-报告初审记录"这样的临时命名

3. **提高容错性**
   - 即使文件编号有误，仍能正确归档
   - 用户可以后续手动修正文件编号
   - 不会因为编号错误导致处理失败

### 命名统一原则

降级后的文件名统一使用目标文件夹名称，原因：

- **简洁明了**：文件名即文件夹名，一目了然
- **避免冗余**：不需要额外记忆文件名规则
- **便于管理**：同一文件夹内的文件名称统一

## ❓ 常见问题

### Q1: 为什么不保留"xxx系统-"前缀？

**A**: 因为没有文件编号，无法确定具体的系统名称。保留"xxx系统-"前缀会导致：
- 文件名不规范（"xxx"是占位符）
- 无法与其他系统级文档区分
- 用户需要手动修改文件名

降级为项目级命名后，文件名清晰明确，便于管理。

### Q2: 降级后如何区分不同系统的文档？

**A**: 降级后的文档归类为项目级文档，不区分系统。如果需要区分：
1. 修正文件编号，重新处理
2. 手动添加系统名称前缀
3. 使用子文件夹分类

### Q3: 能否禁用自动降级？

**A**: 当前版本不支持禁用。如果需要保留系统级命名，建议：
1. 确保所有文档都有正确的文件编号
2. 检查类型简拼是否在 `DOC_TYPE_CODE_MAP` 中
3. 修正OCR识别错误

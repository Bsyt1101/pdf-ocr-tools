# 系统名称自动读取功能

**版本**: v1.5.1
**更新日期**: 2026-03-05

---

## 🎯 功能说明

工具会自动从"项目实施申请单"Word文件中读取系统名称，用于系统级文档的命名。

---

## 📂 文件结构要求

### 目录结构
```
项目目录/
├── 项目文档.pdf                    # 待处理的PDF文件
├── 项目实施单/                     # 必须包含"项目实施单"关键词
│   └── 项目实施申请单.docx         # 必须包含"项目实施申请单"关键词
├── 01测评授权书/
├── 02风险告知书/
└── ...
```

### Word文件格式要求

**项目实施申请单.docx** 必须包含表格，支持以下两种格式：

**格式1：有序号列**
| 序号 | 系统名称 | ... |
|------|---------|-----|
| 1 | 门户网站系统 | ... |
| 2 | 内部办公系统 | ... |

**格式2：无序号列（v1.5.1 新增支持）**
| 系统名称 | 级别 |
|---------|------|
| 门户网站系统 | 二级 |
| 内部办公系统 | 三级 |

**要求**:
- 表格必须有"系统名称"列（"序号"列可选）
- 无序号列时自动按行顺序编号（1, 2, 3...）
- 系统名称不能为空
- 自动跳过「其他要求」「备注」等非系统名称行

## 🔄 工作流程

### 1. 读取系统名称
```
启动工具
  ↓
搜索"项目实施单"文件夹
  ↓
搜索"项目实施申请单"Word文件
  ↓
读取表格中的"序号"和"系统名称"
  ↓
建立映射：{1: "门户网站系统", 2: "内部办公系统", ...}
```

### 2. 识别文档并匹配系统名称（三级回退）
```
识别PDF文档类型
  ↓
如果是系统级文档
  ↓
方式1：从文件编号提取系统序号（如：JDB25300-CPJG-01 中的 01）
  ↓ 序号 → Word映射表查找系统名称
  ↓ 找到？ → 是 → 使用该系统名称
  ↓ 否
  ↓
方式2：将 Word 中所有系统名称与 OCR 文本匹配
  ↓ 逐个检查系统名称是否出现在文本中
  ↓ 找到？ → 是 → 使用该系统名称
  ↓ 否
  ↓
方式3：正则表达式从 OCR 文本内容提取
  ↓ 匹配"XXX系统等级保护"等模式
  ↓ 找到？ → 是 → 使用提取的名称
  ↓ 否 → 使用"xxx系统"占位
```

**重要说明**：
- 文件编号（如 JDB25300-CPJG-01）仅用于识别文档类型和提取系统序号
- 系统序号（01）用于查找对应的系统名称
- 最终文件命名使用的是实际的系统名称，而不是编号
- 例如：01 → 门户网站系统，02 → 内部办公系统

## 💡 使用示例

### 示例1：完整流程

**文件结构**:
```
/项目目录/
├── 广东时代传媒.pdf
├── 项目实施单/
│   └── 项目实施申请单.docx
├── 03测评调研表/
└── 07测评记录及问题汇总/
```

**Word文件内容**:
| 序号 | 系统名称 |
|------|---------|
| 1 | 门户网站系统 |
| 2 | 内部办公系统 |

**PDF文件内容**:
- 第1页：文件编号 JDB25300-CPJG-01（测评结果记录，系统序号01）
- 第2页：文件编号 JDB25300-CPJG-02（测评结果记录，系统序号02）

**处理结果**:
```
读取系统名称:
  序号 1 → 门户网站系统
  序号 2 → 内部办公系统

处理第 1 页:
  识别类型: xxx系统-测评结果记录签字页 (系统级)
  提取文件编号: JDB25300-CPJG-01
  提取系统序号: 01 → 转换为整数 1
  查找映射: 1 → "门户网站系统"
  替换"xxx系统"为"门户网站系统"
  最终文件名: 门户网站系统-测评结果记录签字页.pdf

处理第 2 页:
  识别类型: xxx系统-测评结果记录签字页 (系统级)
  提取文件编号: JDB25300-CPJG-02
  提取系统序号: 02 → 转换为整数 2
  查找映射: 2 → "内部办公系统"
  替换"xxx系统"为"内部办公系统"
  最终文件名: 内部办公系统-测评结果记录签字页.pdf
```

**关键点**：
- 文件编号中的系统序号（01、02）用于查找对应的系统名称
- 最终文件命名使用的是实际的系统名称，而不是编号
- 例如：JDB25300-CPJG-01 → 门户网站系统-测评结果记录签字页.pdf

### 示例2：手动指定系统名称

如果不想使用自动读取功能，可以在运行时手动输入系统名称：

```bash
python main.py

请输入 PDF 文件路径: 广东时代传媒.pdf
请输入项目名称（可选）: JDB25300-广东时代传媒
请输入系统名称（可选）: 门户网站系统  # 手动输入
```

**优先级**: 手动输入 > 自动读取

## ⚠️ 注意事项

### 1. Word文件必须存在
- 如果找不到"项目实施单"文件夹，会显示警告
- 如果找不到"项目实施申请单"Word文件，会显示警告
- 系统级文档会使用"xxx系统"作为前缀（未替换）

### 2. 表格格式要求
- 必须有"系统名称"列（"序号"列可选）
- 无序号列时自动按行顺序编号
- 系统名称不能为空
- 自动跳过非系统名称行（如「其他要求」）
- 如果表格格式不正确，会显示警告

### 3. 文件编号要求
- 系统级文档必须有文件编号（如：JDB25300-DCB-01）
- 文件编号中的系统序号（01, 02, 03...）必须与Word文件中的序号对应
- 如果序号不匹配，会使用"xxx系统"作为前缀

### 4. 依赖库
需要安装 `python-docx` 库：
```bash
pip install python-docx
```

## 🔧 故障排查

### 问题1：未找到项目实施单文件夹
**原因**: 文件夹名称不包含"项目实施单"关键词

**解决**: 确保文件夹名称包含"项目实施单"，例如：
- ✓ `项目实施单`
- ✓ `01项目实施单`
- ✓ `项目实施单（Word+PDF）`
- ✗ `实施单`

### 问题2：未找到项目实施申请单Word文件
**原因**: 文件名不包含"项目实施申请单"关键词

**解决**: 确保文件名包含"项目实施申请单"，例如：
- ✓ `项目实施申请单.docx`
- ✓ `项目实施申请单-2024.docx`
- ✗ `申请单.docx`

### 问题3：未能读取到系统名称
**原因**: Word文件中的表格格式不正确

**解决**: 检查表格格式：
1. 确保有"系统名称"列（"序号"列可选）
2. 有序号列时，序号必须是数字（1, 2, 3...）
3. 确保系统名称不为空

### 问题4：系统名称未替换
**原因**: 三种匹配方式都未成功

**解决**:
1. 检查 Word 文件中是否包含正确的系统名称
2. 检查文件编号格式（如：JDB25300-DCB-01）
3. 确保 OCR 文本中包含系统名称关键字
4. 如以上都无法匹配，手动输入系统名称

# 系统名称一致性验证

**版本**: v1.6.2
**更新日期**: 2026-03-07

## 功能概述

从 v1.6.2 版本开始，工具会自动验证页面提取的系统名称与文件编号中的系统序号是否一致，帮助用户及时发现文件编号或系统名称的错误。

## 验证逻辑

### 验证时机

系统名称一致性验证会在以下两种情况下触发：

1. **模糊匹配时**：当通过文本匹配从实施单中获取系统名称时
2. **内容提取时**：当从页面内容中提取系统名称时

### 验证规则

工具会比较以下信息：
- 文件编号中的系统序号（如 `JDB25300-CPSQS-02` 中的 `02`）
- 实施单中的系统名称映射（如 `{1: "门户网站系统", 2: "内部办公系统"}`）
- 页面提取或匹配到的系统名称

如果发现不一致，会显示警告信息。

## 使用示例

### 场景1：模糊匹配时的不一致

**文件信息**：
- 文件编号：`JDB25300-CPSQS-02`
- 实施单映射：`{1: "门户网站系统", 2: "内部办公系统"}`
- 页面文本：包含"门户网站"关键词

**输出**：
```
  ✓ 从文本匹配系统名称: 门户网站系统 (模糊匹配，序号 1)
  ⚠️  警告：系统名称不一致！
      - 文件编号中的系统序号: 2
      - 匹配到的系统名称: 门户网站系统 (序号 1)
      - 建议检查文件编号或系统名称是否正确
```

**可能原因**：
- 文件编号写错了（应该是 `-01` 而不是 `-02`）
- 页面内容有误（应该是"内部办公"而不是"门户网站"）

### 场景2：内容提取时的不一致

**文件信息**：
- 文件编号：`JDB25300-CPSQS-01`
- 实施单映射：`{1: "门户网站系统", 2: "内部办公系统"}`
- 页面提取：`数据管理系统`

**输出**：
```
  ✓ 从内容提取系统名称: 数据管理系统
  ⚠️  警告：系统名称不一致！
      - 文件编号中的系统序号: 1
      - 实施单中对应的系统名称: 门户网站系统
      - 页面提取的系统名称: 数据管理系统
      - 建议检查文件编号或页面内容是否正确
```

**可能原因**：
- 文件编号写错了（应该是 `-03` 而不是 `-01`）
- 实施单中缺少"数据管理系统"（需要补充）
- OCR 识别错误（实际是"门户网站系统"）

### 场景3：智能比较（不会误报）

**文件信息**：
- 文件编号：`JDB25300-CPSQS-01`
- 实施单映射：`{1: "门户网站系统"}`
- 页面提取：`门户网站平台`

**输出**：
```
  ✓ 从内容提取系统名称: 门户网站平台
```

**说明**：工具会自动去除"系统"/"平台"后缀进行核心名称比较，`门户网站` 与 `门户网站` 匹配，不会触发警告。

## 处理建议

当看到警告信息时，建议按以下步骤处理：

1. **检查文件编号**
   - 确认页面顶部的文件编号是否正确
   - 特别注意系统序号部分（最后两位数字）

2. **检查实施单**
   - 确认项目实施申请单中的系统名称列表是否完整
   - 确认序号与系统名称的对应关系是否正确

3. **检查页面内容**
   - 确认页面中的系统名称是否清晰可读
   - 如果是扫描件，检查 OCR 识别是否准确

4. **手动修正**
   - 如果确认是文件编号错误，可以手动修改文件编号
   - 如果确认是实施单错误，更新实施单后重新运行
   - 如果确认是 OCR 错误，可以尝试使用其他 OCR 引擎

## 技术细节

### 验证代码位置

验证逻辑位于 `src/pdf_processor.py` 的 `process_pdf_with_ocr` 方法中：

- 第 1007-1011 行：模糊匹配时的验证
- 第 1022-1033 行：内容提取时的验证

### 智能比较算法

```python
# 去除常见后缀进行核心名称比较
extracted_core = current_system_name.replace('系统', '').replace('平台', '').strip()
expected_core = expected_name.replace('系统', '').replace('平台', '').strip()

# 只有核心名称完全不匹配时才报警
if extracted_core not in expected_core and expected_core not in extracted_core:
    print("⚠️  警告：系统名称不一致！")
```

### 不触发验证的情况

以下情况不会触发验证：

1. 通过文件编号直接获取系统名称（方式1）
2. 没有文件编号的文档
3. 没有实施单映射表的情况
4. 项目级文档（不需要系统名称）
